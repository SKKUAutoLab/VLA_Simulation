# VLA 자율주행 시스템 기능 보고서

## 0. 전체 아키텍처 — Dual-System (Fast / Slow)

```
[카메라] ─┬─→ (Fast) vla_lora_drive_node : 비전 차선주행 + 기하/라이다 제어 → /topic_control_signal → 차량
          └─→ (Slow) vla_brain_node      : 자연어 해석 + 장면추론(신호등/장애물) → /vla/command
[라이다]  ───→ vla_lora_drive_node (전방 정지/회피 안전계층)
[GUI]     ───→ /nl_command(자연어) · /vla/command(버튼) · 환경 스폰토글
```

- **Fast path(주행정책)** = 비전 LoRA 모델이 차선을 추종(실시간).
- **Slow path(브레인)** = Qwen3-VL-2B가 자유언어 해석 + 카메라 장면추론(제로샷).
- **안전·보조** = 라이다(전방정지/회피), 기하제어(복구·목적지정지·직진).

핵심 설계 원칙: **신뢰성이 필요한 매핑은 2B LLM에 맡기지 않고 결정론적 코드로 처리**(아래 곳곳에서 반복).

## 1. [핵심] 비전 차선주행 — 직진 비틀거림 해결

### 문제
기존 VLA 비전주행이 쉬운 직선에서도 비틀거리고, 한번 밀리면 복귀를 못 했음(직선에서 3m 이탈 후 정지).

### 원인 규명
- 모델 구조: Qwen3-VL-2B 비전 → 토큰(7행×10열 공간그리드, 2048-d) → **`mean(1)` 풀링** → WP 헤드 → 6개 웨이포인트 → pure-pursuit 조향.
- **`mean(1)`이 토큰의 공간정보(특히 가로=차선의 좌우 치우침)를 통째로 평균내 소거**. 차선유지의 본질인 "횡방향 위치"를 입력에서 잃어버림. → 직선에서 횡오차가 헤딩오차로 안 드러나(제어식 `he=atan2(ey,ex)`만 사용) 보정신호가 안 생김. DAgger·수동시연 데이터를 다 줘도 정보가 지워지니 못 씀.

### 해결 (학습)
- **공간보존 헤드**로 교체: 토큰별 Linear(2048→64) + 위치임베딩 + 전체 flatten→512 + FiLM(차선 조건부) → WP. 공간정보를 헤드에 그대로 전달.
- **학습 방식**: 비전은 동결(frozen base Qwen vision), 토큰특징을 1회 캐시 → **헤드만 학습**(`train_head_fast.py`, 9초/에폭). 데이터=수동 텔레op 시연 + 텔레포트 복구 WP + DAgger.
- **결과**: wpMAE 0.56(mean-pool)→**0.284**(공간헤드+양방향). 실주행 **직선 차선유지 중앙 0.08~0.10m, 커브 최대 0.85m, 안전망 개입 0회**(이전 3m 이탈 대비 30배 개선).

### 동작
카메라프레임 → Qwen비전 토큰(공간보존) → 공간헤드 → 6 WP(ego좌표) → pure-pursuit(룩어헤드 0.55m, GAIN 13)로 조향. FiLM의 lane 임베딩으로 1/2차선 선택.

> 교훈: PilotNet(CNN)이 됐던 이유도 conv가 공간을 안 뭉갰기 때문. 풀링이 차선유지의 진짜 병목이었음.

### 1-1. 이미지 → 웨이포인트 생성 (상세)

**웨이포인트 정의**: 6개 점 `(ex, ey)`, ego 좌표계(ex=전방, ey=좌, m), WP_SCALE=5.0 정규화. 의미 = "이 시야에서 앞으로 따라갈 경로 6점".

**학습 라벨 생성** (`generate_wp_data.py`, 텔레포트 GT):
1. 차를 차선 중앙선 점 + **횡오프셋(±0.9m)·헤딩변형(±15°)** 자세로 텔레포트
2. 그 자리에서 카메라 이미지 캡처
3. 라벨 = 그 포즈 기준 **미래 중앙선점 6개를 ego 좌표로 투영**(월드점→차 전방/좌 회전변환)
4. → `(이미지, 6 WP)` 쌍. + 사람 텔레op 시연(`manual_wp_labels.csv`) 동일 포맷.
   - 핵심: 오프셋·헤딩을 흔들어 찍어 "밀렸을 때 중앙복귀 WP"까지 학습 → off-center 복구.

**추론 파이프라인** (`_feat` → `Head.forward`):
```
카메라 BGR
 → Qwen processor(320×240) → pixel_values(280패치)
 → Qwen 비전 인코더 → 토큰(70,2048)            # 7×10 공간그리드 ★풀링 안 함★
 → view(1,70,2048)
 ── Head ──
 → 토큰별 Linear(2048→64) + 위치임베딩          # 공간위치 유지
 → flatten(1,4480) → Linear(4480→512)
 → FiLM(lane): h = h·(1+γ)+β                    # 1/2차선 조건부
 → MLP(512→256→12) → 6×(ex,ey) × 5.0           # 웨이포인트
```
핵심 = 풀링 없이 토큰 공간배치(가로=차선 좌우위치)를 헤드까지 전달 → 횡오차를 WP에 반영.

**WP → 조향(pure-pursuit)**: 첫 ex≥0.55m WP 선택 → `he=atan2(ey,ex)` → `steer=clamp(-he·13, ±7)`(EMA). 모델은 *경로*만 예측, 조향각은 기하로 계산(과소조향 회피).

## 2. 양방향 주행 (정방향 CCW / 역방향 CW)

### 학습
- 기존 데이터는 전부 한 방향(중앙선 +index = 시계방향 CW = "역방향")만 있었음.
- `generate_wp_data.py`에 `FWD=1` 모드 추가: 차를 반대로 돌려세워(yaw+π) 역순 인덱스로 미래 WP 생성 → 정방향(CCW) 데이터 10,080장 추가.
- 양방향 데이터(역 10,080 + 정 10,080 + DAgger + 수동)로 헤드 재학습 → 모델이 차의 시야(방향)로 따라갈 방향을 자동 판별.

### 동작
- `ddir` 상태(+1=CW역방향, **-1=CCW정방향, 기본값**).
- 아커만 차량은 조향으로 U턴 불가 → **방향전환은 텔레포트 스냅(`복구`)으로** 처리. 명령에 "정방향/역방향/반대로"가 있으면 그 방향으로 스냅 후 비전추종.
- 같은 이미지라도 차가 어느 쪽을 보느냐로 비전이 방향을 결정(별도 입력 불필요).

## 3. 목적지 정지 ("횡단보도에서 정지" 등)

### 구현(학습 아님 — 기하+결정론)
- 랜드마크 월드좌표(`track_data.py`): 출발점·횡단보도/신호등·중간·주차구역.
- drive 노드: `<랜드마크>+정지` → 그 차선의 랜드마크 최근접 인덱스를 목표로 설정(`stop_at_idx`), 비전으로 차선 주행하다 진행거리 누적 후 목표 인덱스 근접 시 정지.
- **brain은 결정론적 우회**: 랜드마크+정지 키워드면 2B LLM 거치지 말고 원문을 drive 노드로 직통(2B가 "신호등"·영어를 즉시정지로 흘리는 불안정 회피). → 한·영·동의어 모두 안정.

---

## 4. 직진(차선무시) 모드 ("신호등으로 직진")

### 구현(기하)
- drive 노드: `<랜드마크>+직진/곧장/차선무시` → `direct_target`(랜드마크 좌표) 설정 → **차선 무시하고 odom 기반 pure-pursuit로 직선 접근**(비전보다 우선), 도달 시 정지.
- brain 결정론적 직통(목적지정지와 동일 방식).
- 검증: 바닥(-2.5,-22)→신호등(-5.6,17.9)을 주차장 가로질러 직선으로(차선추종이면 트랙을 돌았을 것).

## 5. 라이다 전방정지 · 회피 (안전계층)

### 하드웨어 수정
- 라이다가 **후방 장착**이라 차체가 전방빔을 가림 → 전방 감지 불가.
- `model.sdf`에서 라이다를 **전방으로 이식**(y -2.6). 단순 이동 시 고정조인트가 튕겨 날아감 → **gravity=0 + 관성 COM 원점화**로 안정 부착.
- 360° 스캔, 보정값: 차 전진 = **스캔 268°**(차체가림은 후방 43~136°, 안 겹침).

### 동작(학습 아님 — 결정론)
- `_scan`: 전방아크(268°±22°) 최소거리 < 2.5m면 **비상정지**(조향유지·속도0).
- **회피**: 전방 6m에서 미리 다른차선 앞쪽 라이다 클리어런스 확인 → 트였으면 그 차선으로 1회 변경(완만 조향 ±4·저속), 양쪽 막히면 정지. 플립플롭 방지 플래그.
- 기존 brain의 LLM 스웨브(꺾어서 옆차 박음)를 라이다 회피가 대체.

## 6. 신호등 · 자연어 (브레인, 제로샷)

### 동작(학습 아님 — Qwen3-VL-2B 제로샷)
- **장면추론**(1.5Hz): 카메라로 `HAZARD=none|redlight|obstacle` 제로샷 판단 → 빨간불=일시정지/초록불=재개. (장애물은 라이다에 위임)
- **자연어 해석**: 자유언어 → `LANE/ACT/LAPS/STOP_AT` 스키마. 한국어·영어 모두.
- **결정론적 보정**(2B 흔들림 방지): "forever/계속"→무한, 숫자없는 "돌고 멈춰/and stop"→1바퀴, "stop만"→즉시정지, 명시숫자 유지. (2B가 "go around and stop"을 무한으로 오인하던 문제 해결)

## 핵심 요약 — "어떻게 동작 가능하게 했나"

1. **학습으로 푼 것**: 직선주행(공간보존 헤드 = 진짜 병목 제거), 양방향(역방향 데이터 생성+재학습). → 비전 모델 자체를 고침.
2. **기하/결정론으로 푼 것**: 복구·목적지정지·직진·라이다 정지/회피·방향전환 스냅. → 2B의 불안정을 코드로 대체(신뢰성).
3. **제로샷으로 푼 것**: 신호등 판단·자연어 해석. → 학습 없이 Qwen3-VL-2B 활용 + 결정론적 보정으로 안정화.

운영: 전체 재현 = `ros2 launch lora_pipeline/vla_drive.launch.py`.
