# 설계서 — (준)순수 단일 VLA 주행 시스템

> 연구(Discover) 결론 기반 Define 문서. 환경: RTX 5090 Laptop 24GB, Gazebo 2D 저속 트랙, ≥10FPS.
> 핵심 원칙: **하나의 소형 VLA 체크포인트가 카메라+언어명령 → 웨이포인트(궤적)를 1-pass로 출력**하고,
> 사소한 pure-pursuit 추종기만 조향으로 변환. (SimLingo 청사진 + SmolVLA 실시간 기법)

## 1. 목표 / 범위
- **G1 차선 추종**: 카메라만으로 지정 차선(1/2)을 유지 주행.
- **G2 언어 명령**: "신호등으로 가줘 / 직진해서 / 2차선으로 / 멈춰" 등을 한 모델이 해석·수행.
- **G3 미학습 객체 반응**: 사전학습 없는 주차차량/신호등/주차를 VLM prior로 판단해 정지·회피 등 행동.
- **비범위(후순위)**: 정밀 주차 기동, 다차량 상호작용.

## 2. 아키텍처 (단일 체크포인트)
```
카메라(전방) + 명령어 텍스트
   │
   ▼
[소형 VLM 백본]  (동결 + LoRA, 개방어휘 지각 보존)
   │  1 forward pass (비자기회귀)
   ├─▶ 웨이포인트 헤드: 미래 N개 (경로점 + 속도) 청크  ← 주 출력
   └─▶ (선택) 언어 헤드: 상태/이유 출력 (해석성)
   │
   ▼
pure-pursuit + 속도 PID  (수십 줄, 학습 X)
   │  비동기: VLM 저빈도(5~10Hz), 추종기 고빈도 실행
   ▼
topic_control_signal (steering, speed)
```
- **"순수성" 근거**: 지각·언어·계획이 모두 학습된 한 모델 안. 외부는 기하 추종기뿐(고전 perception/decision 아님).

## 3. 컴포넌트 스펙
| 항목 | 결정 | 근거 |
|---|---|---|
| 백본 | Qwen3-VL-2B(현행) 시작 → 필요시 Qwen2.5-VL-3B / SmolVLM2-500M | 이미 통합·실측 11FPS, 처음부터 소형(연구 B결론) |
| 백본 학습 | **동결 + LoRA(r16~32)** + 일반 VQA 소량 공동학습 | 개방어휘 망각 방지(π0.5 knowledge insulation, VLM2VLA) |
| 행동헤드 | 웨이포인트 회귀(경로 N점 + 속도) — 토큰조향 ❌ | 토큰조향은 느리고 부정확(FAST/OpenVLA-OFT) |
| 청크 길이 | N=5~10 (저속이라 충분), 비동기 실행 | SmolVLA async+chunking |
| 변환 | pure-pursuit(횡) + 속도PID(종) | LMDrive/표준 |
| 입력 해상도 | 320×240 시작 (FPS), 필요시 ↑ | 실측 11FPS 지점 |

## 4. 데이터 스펙 (정답값 기반 — 사용자 원칙)
- **차선 GT**: `track_data.py TRACK_CENTERLINE`(1차선) + `LANE_SEPARATION=2.5m` infield쪽(2차선, `build_lane2_centerline.py`).
- **수집**: 중앙선을 **연속 traversal**(set_entity_state, dl=0 접선각)로 이미지 + 정답 웨이포인트(미래 중앙선 점들) 라벨. **노이즈 금지.**
- **복구(필수, 노이즈 아님)**: 중앙선 근처 **소량 오프셋 포즈**(±0.3~0.6m, ±10°)에서 **정답=중앙선 복귀 궤적**. → covariate shift 해결(모방학습 필수). *주: 사용자와 "이건 노이즈가 아니라 오프셋 위치의 정답"임을 합의 필요.*
- **언어 페어링**: 각 클립에 명령 텍스트("1차선 유지","신호등까지","정지"). SimLingo **Action Dreaming**식 명령↔궤적 증강(명령수행 SR 28→73).
- **미학습객체**: 정지/회피 데모 소량 + 백본 prior로 일반화.
- 포맷: 이미지 + (웨이포인트, 속도, lane_id, command) CSV/parquet.

## 5. 학습 계획
1. 백본 동결, LoRA + 웨이포인트 헤드만 학습(우리 CNN→헤드로 진화).
2. 일반 VQA 소량 공동학습으로 개방어휘 보존.
3. 손실: 웨이포인트 L1/L2 회귀 (+ 선택 언어 CE).
4. 단일 24GB에서 수 시간(SmolVLA ~4h 참고).

## 6. 실시간(≥10FPS) 로드맵
1. 소형 백본 + 웨이포인트 청크 + **비동기**(최우선) → 3~10Hz VLM이 30Hz+ 제어로.
2. 저해상도/비주얼토큰 감축.
3. 양자화: Blackwell **FP8**(또는 INT4-Marlin), TensorRT-LLM(배치1 저지연).
4. terse 출력. KV캐시(정적 프롬프트 prefix 재사용).
- 저속 보너스: 5Hz도 안정 가능(스텝당 수 cm).

## 7. 마일스톤 (점진)
- **M1**: 1차선 차선유지(언어=고정) — 웨이포인트 헤드+pure-pursuit, GT 연속데이터. (수용: 한 바퀴 이탈<lane폭/2, 복귀 가능)
- **M2**: 차선 명령("1/2차선") 조건부 — lane_id 입력. (수용: 지정차선 점유율>90%)
- **M3**: 목표 명령("신호등까지/직진") — 목표도달 정지. (수용: REACHED, 양 모드 동작)
- **M4**: 미학습객체("앞 차 서면 정지") — VLM prior 반응. (수용: 주차차량 앞 정지)
- **M5**: 비동기+양자화로 ≥10FPS 확정 + 시연 녹화.

## 8. 수용 기준(공통)
- closed-loop 한 바퀴 완주, 도로 이탈 평균<1.5m.
- 잘못된 역주행/상수붕괴 없음.
- publisher=1(잔존노드 점검), 실주행 검증 후에만 "성공" 표기.

## 9. 리스크 / 완화
| 리스크 | 완화 |
|---|---|
| 순수 중앙선만 학습→드리프트 복구 불가 | 오프셋 포즈 정답(복구) 데이터 포함(노이즈 아님) — 사용자 합의 |
| 백본 파인튜닝→개방어휘 망각 | 동결+LoRA+VQA 공동학습 |
| VLM 정밀 조향 약함 | 조향은 헤드+pure-pursuit, VLM은 계획/의미 |
| FPS 부족 | 비동기+청킹+FP8/TensorRT |
| 잔존 ROS 노드 거짓성공 | 실주행 전 publisher count 확인 |

## 10. 기존 자산 재사용 맵
- `train_cnn(_lane).py` PilotNet → **웨이포인트 헤드**로 진화(출력 N점).
- `vla_cnn_drive_node.py` → 백본+헤드 통합 추론노드로.
- `generate_gt_data.py` + `build_lane2_centerline.py` → GT 연속수집(+오프셋 정답).
- `track_data.py` → 차선/목표/주차 GT 단일 출처.
- 명령파서(Qwen) → 백본이 직접 명령 해석(통합).
